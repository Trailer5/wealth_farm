# 数据获取底层接口状态

本文档记录数据获取底层接口的当前状态、停用原因、压力风险和后续替代方向。

本文档只记录底层接口状态。本文档不记录短期开发计划。

## 总体结论

当前主线能力不依赖下列不稳定接口。

当前主线能力已经覆盖：

1. A 股日线和周线。
2. ETF 实时与日线。
3. 场内基金东方财富、腾讯、新浪实时数据。
4. 场外基金净值、申购赎回状态、估值、规模、行业配置。
5. A 股三大报表、财务指标和财报披露计划。
6. CNINFO、SSE、SZSE、BSE/CNINFO、中证基金、基金公司、证监会公开信息。
7. PDF 下载、PDF 文本/表格解析、OCR 队列、财报字段抽取。

以下接口不得作为当前主线功能依赖。

## 已停用接口

### `AkShareDataSource.get_a_share_spot`

状态：停用。

底层接口：`stock_zh_a_spot_em`。

问题：温和模式三次真实调用仍出现远端断开连接。

压力风险：该接口访问全市场实时行情，单次请求会触发 AkShare 内部分页访问。访问量偏大时容易被上游关闭连接。

当前替代：项目当前最低要求是日线级别，A 股日线和周线已可用。

后续方向：需要实时行情时，必须新增独立的温和实时行情 provider，不得直接恢复该 AkShare wrapper 作为主线。

### `AkShareDataSource.get_a_share_minute_bars`

状态：停用。

底层接口：`stock_zh_a_hist_min_em`。

问题：温和模式三次真实调用仍出现远端断开连接。

压力风险：分钟线接口对上游请求频率、时间窗口和参数格式更敏感。

当前替代：项目当前最低要求是日线级别，A 股分钟线不属于当前基线能力。

后续方向：需要分钟级研究时，必须独立设计分钟线数据源、限流和缓存，不得把该接口直接并入主线。

### `AkShareDataSource.get_industry_board_constituents`

状态：停用。

底层接口：`stock_board_industry_cons_em`。

问题：温和模式三次真实调用仍出现东方财富远端断开连接。

压力风险：该接口会重新访问行业板块映射和成分接口，容易叠加请求压力。

当前替代：行业板块列表已经可用；当前主线不依赖行业成分股。

后续方向：需要行业成分时，必须新增独立东方财富板块成分 provider，并做缓存和低频刷新。

### `BaoStockDataSource.get_all_stocks`

状态：停用。

底层接口：`query_all_stock`。

问题：独立子进程 90 秒超时并退避重试后仍未返回。

压力风险：该接口一次性请求全市场列表，服务端响应不稳定时会阻塞整个验证或任务进程。

当前替代：CNINFO 证券映射、单票 `query_stock_basic`、后续交易所/其他 provider 列表可以覆盖证券范围。

后续方向：需要全市场证券池时，优先使用 CNINFO 映射或交易所列表，不得把 BaoStock 全市场列表作为主线依赖。

## 保留但不稳定接口

### `AkShareDataSource.get_fund_stock_holdings`

状态：保留实验入口，不作为主线依赖。

底层接口：`fund_portfolio_hold_em`。

问题：温和模式三次真实调用仍出现 AkShare 上游解析 `JSONDecodeError`。

独特价值：基金股票持仓可以提供基金穿透研究价值。

当前替代：优先从基金定期报告、CNINFO、中证基金披露和基金公司公告 PDF 中解析持仓。

后续方向：如果需要基金持仓，必须优先建设官方披露 PDF 结构化解析；仅在确认 AkShare 修复后再考虑恢复该入口。

### `AkShareDataSource.get_fund_bond_holdings`

状态：保留实验入口，不作为主线依赖。

底层接口：`fund_portfolio_bond_hold_em`。

问题：温和模式三次真实调用仍出现 AkShare 上游解析 `JSONDecodeError`。

独特价值：基金债券持仓对债基和固收类产品研究有价值。

当前替代：优先从基金定期报告、CNINFO、中证基金披露和基金公司公告 PDF 中解析持仓。

后续方向：如果需要债券持仓，必须优先建设官方披露 PDF 结构化解析；仅在确认 AkShare 修复后再考虑恢复该入口。

### `EastmoneyResearchReportDataSource.download_pdf`

状态：保留显式 opt-in 入口，不作为主线依赖。

底层接口：东方财富研报 PDF URL 模板。

问题：真实下载样本返回 `Content-Type: application/pdf`，但文件头不是 `%PDF`，内容为脚本页。

独特价值：研报 PDF 原文对后续研报正文分析有价值。

当前替代：东方财富研报元数据已经可用；正文分析暂不作为当前数据底座基线。

后续方向：需要研报正文时，必须研究东方财富真实下载机制、寻找其他研报 PDF 源，或接入合法可下载的研报来源。

## 压力与访问规则

真实外部验证必须优先使用温和模式。

推荐命令：

```shell
python src/smoke_tests/run_real_data_validation.py --gentle --case-timeout 120 --retries 2 --retry-delay 20
```

对疑似全市场、分页或高压力接口，必须单项验证，不得整组高频重复运行。

真实验证不得把“调用过”当作完成。只有返回非空真实数据，下载类文件头确认为 `%PDF`，才可以记录为通过。
