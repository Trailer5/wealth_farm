# 短期开发计划：数据源与数据库

## 目标

本阶段建立可落地的数据获取底座，覆盖 A 股行情、基金数据、官方披露财报、PDF 处理和 SQLite 存储。

本阶段优先保证日线级别数据可用。本阶段不追求交易级实时行情。

## A 股行情

A 股行情使用 BaoStock 与 AkShare 的组合。

BaoStock 作为 A 股历史行情和基础财务数据的稳定主源。

AkShare 作为实时行情、补充字段、指数、行业和异常数据的扩展源。

短期任务：

1. 封装 BaoStock 日线数据适配器。
2. 封装 AkShare A 股行情适配器。
3. 建立统一股票代码格式转换规则。
4. 建立日线数据表结构。
5. 建立双源字段对账机制，先覆盖开盘价、收盘价、最高价、最低价、成交量、成交额。

## 基金数据

基金数据分为场内基金和场外基金。

### 场内基金

场内基金按照证券行情处理。

场内基金开发期同时接入东方财富、腾讯、新浪和 AkShare 多个数据源。

开发阶段保留多个数据源用于对比稳定性、字段完整性和延迟表现。正式使用时再决定保留和删除哪些来源。

短期任务：

1. 建立 ETF、LOF、封闭式基金的代码识别规则。
2. 封装东方财富场内基金行情适配器。
3. 封装腾讯场内基金行情适配器。
4. 封装新浪场内基金行情适配器。
5. 封装 AkShare 场内基金行情适配器。
6. 建立场内基金日线表结构。

### 场外基金

场外基金不按实时交易行情处理。

场外基金以基金代码为主键，优先获取每日单位净值、累计净值、日增长率、申购状态、赎回状态、基金规模、持仓和资产配置。

场外基金正式净值以基金公司、监管指定披露平台、天天基金/东方财富等公开数据为主要来源。

场外基金估值只能作为参考数据。场外基金估值不得等同于正式净值。

推荐方案：

1. 使用 AkShare 封装的东方财富/天天基金接口获取开放式基金每日净值和历史净值。
2. 使用基金公司官网和监管披露平台校验关键定期报告。
3. 使用基金代码统一关联支付宝可购买基金、天天基金基金档案和官方披露文件。
4. 将净值数据、基金档案、持仓披露和资产配置分表存储。
5. 将平台估值字段单独存储，并标注为 `estimated`。

短期任务：

1. 建立场外基金基础信息表。
2. 建立场外基金净值表。
3. 建立场外基金披露文件索引表。
4. 建立场外基金估值字段的隔离存储规则。

## 财报与官方披露

财报和审计资料必须使用官方披露源。

A 股上市公司披露源优先级：

1. 巨潮资讯网。
2. 上交所、深交所、北交所官网。
3. 证监会公开信息。
4. 公司官网投资者关系页面。

基金披露源优先级：

1. 巨潮资讯网。
2. 上交所、深交所基金公告页面。
3. 中证网基金信息披露平台。
4. 基金公司官网。

落地方案：

1. 建立公告检索适配器，按证券代码、基金代码、报告类型和日期范围查询公告。
2. 建立披露文件索引表，记录标题、发布日期、公告类型、来源、原始 URL、文件 URL 和下载状态。
3. 下载 PDF 原文到 `data/` 下的文件库。
4. 对 PDF 计算 hash，避免重复下载和重复解析。
5. 抽取 PDF 文本并保存解析结果。
6. 对年报、半年报、季报、审计报告建立分类标签。
7. 对解析失败的 PDF 记录失败原因，并进入后续 OCR 队列。

PDF 表格抽取备选：

## SQLite 数据库

数据库使用 SQLite。

SQLite 文件必须落入 `data/database/`。

短期建议表：

1. `securities`：证券与基金基础信息。
2. `stock_daily_bars`：A 股日线行情。
3. `fund_exchange_daily_bars`：场内基金日线行情。
4. `fund_nav_daily`：场外基金每日净值。
5. `fund_estimates`：场外基金估值数据。
6. `disclosure_documents`：公告和披露文件索引。
7. `document_files`：PDF 原文文件记录。
8. `document_texts`：PDF 文本解析结果。
9. `source_checks`：多源对账记录。

短期数据库原则：

1. 所有数据必须记录 `source`。
2. 所有外部文件必须记录原始 URL。
3. 所有 PDF 必须记录文件 hash。
4. 所有行情和净值数据必须记录交易日或净值日期。
5. 所有估值数据必须标注为估计值。

## 开发步骤拆解

### 第一步：行情与基础数据底座

完成 A 股日线、场内基金行情、场外基金净值、基础信息和估值隔离。

本步骤的目标是让项目具备稳定的日线级行情数据、基金基础数据和多源对账能力。

### 第二步：官方披露与 PDF 文件链路

完成官方公告检索、PDF 下载、hash 去重、披露文件索引、PDF 文本抽取和 PDF 表格抽取。

本步骤的目标是让项目具备可追溯的官方披露资料获取能力。

### 第三步：基金披露与配置数据补全

补齐基金公司官网、中证网基金信息披露平台、基金持仓、资产配置、行业配置和多源校验。

本步骤的目标是让基金研究不仅依赖销售平台数据，也能追溯到正式披露资料。

### 第四步：文档解析增强

建立 OCR 队列，接入 OCR 引擎，并在 PDF 表格和文本基础上抽取标准财务字段。

本步骤的目标是从“文档级数据”推进到“指标级数据”。

### 第五步：数据质量与自动化

建立真实网络 smoke test、schema 版本记录、数据源稳定性评分和自动降级策略。

本步骤的目标是让数据底座从可用走向可持续维护。
